# Модуль суммаризации статей

## Модель

**cointegrated/rut5-base-absum** — русскоязычная seq2seq-модель на базе архитектуры **T5-base** (Text-to-Text Transfer Transformer, Google, 2020). Оригинальная T5 была предобучена на английском корпусе C4 методом маскированного восстановления текста; данная версия была дополнительно дообучена на русскоязычном датасете новостных статей Газеты.ру (Gazeta), что адаптировало её к задаче абстрактивной суммаризации на русском языке. Модель имеет ~250 млн параметров и принимает на вход текст, возвращая перефразированное краткое содержание (не простую вырезку предложений).

---

## Дообучение

**Дообучение (fine-tuning)** — процесс дополнительного обучения предобученной модели на узкоспециализированном датасете для адаптации к конкретной предметной области. В данном случае базовая модель, обученная на новостных текстах, дообучается на корпусе IT-статей платформы EduFlow: это позволяет модели усвоить характерную лексику (технические термины, названия фреймворков, аббревиатуры) и стиль изложения, типичный для IT-тематики, что повышает качество генерируемых саммари по сравнению с базовой моделью.

---

## Redis — кэширование саммари

Redis используется для хранения уже сгенерированных кратких содержаний, чтобы не запускать дорогостоящий инференс модели повторно. Саммари хранятся по ключу `summary:article:{id}` со временем жизни **30 дней**; политика вытеснения — **allkeys-lfu** (Least Frequently Used), то есть при нехватке памяти (лимит 256 MB) Redis автоматически удаляет наименее часто запрашиваемые записи. При обновлении или удалении статьи её саммари немедленно инвалидируется через DELETE-запрос, чтобы следующий запрос получил актуальное содержание.

## Параметры дообучения

Обучение запускается на MacBook M1 Pro 13" 8 GB с MPS-ускорением.

### Ключевые параметры

Параметры, которые напрямую влияют на качество и поведение обучения.

| Параметр | Значение | Назначение |
|---|---|---|
| **Скорость обучения** (Learning Rate) | **5e-5** | Наиболее критичный параметр. Определяет, насколько сильно модель обновляет веса на каждом шаге. Значение 5e-5 — стандарт для fine-tuning T5: достаточно мало, чтобы не разрушить предобученные веса, и достаточно велико для сходимости на небольшом датасете. |
| **Планировщик скорости обучения** (LR Scheduler) | **cosine** | Задаёт закон изменения learning rate по ходу обучения. Cosine-расписание плавно снижает lr к нулю по косинусной кривой — на небольших датасетах даёт лучшую финальную точку, чем линейный спад. |
| **Прогрев** (Warmup) | **10% шагов** | Первые 10% шагов lr линейно растёт от 0 до целевого значения. Предотвращает нестабильность градиентов в самом начале обучения, когда модель ещё не адаптирована. |
| **Количество эпох** (Epochs) | **7** (макс.) | Число полных проходов по обучающей выборке. При датасете 1 000–1 500 статей (800–1 200 train-примеров) и эффективном батче 16 одна эпоха составляет 50–75 шагов — 5 эпох может не хватить для выхода на плато, поэтому лимит повышен до 7; фактически обучение завершается раньше благодаря ранней остановке. |
| **Ранняя остановка** (Early Stopping) | **patience = 3** | Если ROUGE-2 на eval-выборке не улучшается 3 эпохи подряд — обучение завершается автоматически и сохраняется лучший чекпоинт. Значение 3 (вместо 2) выбрано под объём 1 000–1 500 статей: при таком размере датасета модели нужно больше итераций для стабилизации, и досрочная остановка после 2 эпох могла бы прервать обучение в ещё неоптимальной точке. |
| **Регуляризация весов** (Weight Decay) | **0.01** | L2-штраф за большие веса. Лёгкое значение 0.01 снижает риск переобучения на небольшом датасете без заметного замедления сходимости. |

### Технические параметры

Параметры, обусловленные ограничениями оборудования и архитектурой задачи.

| Параметр | Значение | Назначение |
|---|---|---|
| **Размер батча** (Batch Size) | **2** | Физическое число примеров за один шаг. Значение 2 — из-за ограничения unified memory 8 GB: модель (~1 GB) + состояния оптимизатора (~2 GB) + активации не должны превысить лимит. |
| **Накопление градиентов** (Gradient Accumulation) | **8 шагов** | Градиенты накапливаются 8 шагов перед обновлением весов. Эффективный батч = 2 × 8 = **16** — компенсирует маленький физический батч. |
| **Клиппинг градиентов** (Gradient Clipping) | **1.0** | Если норма градиента превышает 1.0, он масштабируется вниз. Стандартный приём для seq2seq-моделей — предотвращает взрывной рост градиентов. |
| **Контрольные точки градиентов** (Gradient Checkpointing) | **включено** | Не хранит промежуточные активации в памяти, пересчитывает их при обратном проходе. Экономит ~30% памяти за счёт дополнительных вычислений. |
| **Максимальная длина входного текста** | **512 токенов** | Стандартный лимит T5-base. Статьи длиннее обрезаются при токенизации. |
| **Максимальная длина саммари** | **200 токенов** | Ограничение на длину генерируемого краткого содержания (~150 слов). |
| **Лучи при генерации** (Beam Search) | **4** | Число параллельных гипотез при генерации на eval. 4 луча дают хороший баланс между качеством и скоростью. |
| **Смешанная точность** (Mixed Precision) | **отключено** | fp16/bf16 отключены: MPS (Apple Silicon) не поддерживает их надёжно — обучение идёт в float32. |

## Датасет

| Параметр | Значение |
|---|---|
| Общее количество статей | **~1 000** (минимальный случай) / **~1 500** (хороший случай) |
| Train (обучение) | **80%** |
| Eval (валидация по эпохам) | **10%** |
| Test (финальная оценка) | **10%** |

Разбивка **80/10/10** выбрана как стандартная для небольших датасетов: большая доля отдаётся обучению, при этом eval-выборки достаточно для отслеживания ROUGE-2 по эпохам и ранней остановки, а test остаётся изолированным для честного итогового сравнения с базовой моделью.

**Формат датасета — JSONL** (один JSON-объект на строку):

```jsonl
{"text": "Полный текст IT-статьи...", "summary": "Краткое содержание статьи в 2-4 предложениях."}
{"text": "Docker — это платформа для контейнеризации приложений...", "summary": "Docker позволяет упаковывать приложения в изолированные контейнеры, обеспечивая воспроизводимость среды запуска."}
```

## Метрики оценки качества

### ROUGE-1

**Что показывает:** долю совпадающих отдельных слов (юниграмм) между сгенерированным саммари и эталонным. Отражает, насколько модель верно воспроизводит ключевую лексику оригинала.

**Диапазон значений:** 0 — 1. Для абстрактивной суммаризации на русском языке типичный рабочий диапазон: 0,20–0,45; значения выше 0,35 считаются хорошим результатом.

---

### ROUGE-2

**Что показывает:** долю совпадающих биграмм (пар соседних слов) между гипотезой и эталоном. Оценивает точность воспроизведения словосочетаний и фразовых конструкций, характерных для предметной области. Является наиболее чувствительной к доменной адаптации метрикой — именно по ней лучше всего заметен эффект дообучения.

**Диапазон значений:** 0 — 1. Для абстрактивной суммаризации: 0,05–0,22; значения выше 0,14 считаются хорошим результатом.

---

### ROUGE-L

**Что показывает:** длину наибольшей общей подпоследовательности (Longest Common Subsequence, LCS) между гипотезой и эталоном, нормированную на длины обоих текстов. Улавливает структурное сходство и связность: метрика ненулевая даже при перестановке слов, если общая цепочка сохраняется.

**Диапазон значений:** 0 — 1. Как правило, располагается между ROUGE-1 и ROUGE-2; для русскоязычной абстрактивной суммаризации: 0,17–0,38.

---

### BERTScore F1

**Что показывает:** семантическое сходство между гипотезой и эталоном, вычисленное через попарные косинусные расстояния между контекстными эмбеддингами токенов (на базе multilingual BERT). В отличие от ROUGE, не требует точного совпадения слов — улавливает смысловую близость синонимов и парафраз, что особенно важно для абстрактивных моделей.

**Диапазон значений:** Практический рабочий диапазон для русского языка: 0,78–0,92. Метрика не обнуляется даже для случайного текста из-за природы пространства эмбеддингов; разница в 0,02–0,03 единицы считается статистически значимой.

---

### Repetition Rate (3-gram)

**Что показывает:** долю повторяющихся триграмм (трёхсловных последовательностей) среди всех триграмм сгенерированного текста. Высокое значение указывает на «зацикливание» модели — многократное воспроизведение одних и тех же фрагментов, что критически снижает читаемость саммари. *Чем ниже значение, тем лучше.*

**Диапазон значений:** 0 — 1. Качественные саммари: < 0,05; значения 0,05–0,15 — заметное зацикливание; > 0,15 — серьёзная проблема генерации.

## Сравнение базовой и дообученной модели

Базовая модель — **cointegrated/rut5-base-absum** без дополнительного обучения, применённая к IT-статьям. Поскольку модель предобучена на корпусе новостей Газеты.ру, её применение к IT-тематике является задачей вне обучающего домена, что закономерно снижает метрики по сравнению с in-domain использованием.

### Таблица 1 — Минимальный случай

Сценарий: дообучение завершилось по ранней остановке после 2–3 эпох, датасет небольшой или частично низкого качества. Улучшения наблюдаются по всем метрикам — дообучение можно считать состоявшимся, однако прирост умеренный.

| Метрика | Базовая модель | Дообученная модель | Прирост |
|---|---|---|---|
| **ROUGE-1** | 0.264 | 0.291 | +10.2% |
| **ROUGE-2** | 0.094 | 0.109 | +16.0% |
| **ROUGE-L** | 0.221 | 0.244 | +10.4% |
| **BERTScore F1** | 0.804 | 0.821 | +2.1% |
| **Repetition Rate (3-gram) ↓** | 0.089 | 0.063 | −29.2% |

### Таблица 2 — Хороший случай

Сценарий: обучение прошло стабильно, датасет достаточного объёма и качества, модель успешно усвоила IT-лексику и стиль. Типичный ожидаемый результат при корректно настроенном дообучении.

| Метрика | Базовая модель | Дообученная модель | Прирост |
|---|---|---|---|
| **ROUGE-1** | 0.264 | 0.323 | +22.3% |
| **ROUGE-2** | 0.094 | 0.131 | +39.4% |
| **ROUGE-L** | 0.221 | 0.277 | +25.3% |
| **BERTScore F1** | 0.804 | 0.843 | +4.9% |
| **Repetition Rate (3-gram) ↓** | 0.089 | 0.039 | −56.2% |

> **Примечание по Repetition Rate:** стрелка ↓ означает, что для данной метрики снижение является улучшением. Прирост указан как уменьшение относительно базового значения.
